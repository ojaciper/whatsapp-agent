================================================================================
                        NEXT STEPS TO RUN
================================================================================

1. CONFIGURE ENVIRONMENT VARIABLES:
   cp .env.local.example .env.local
   
   Then edit .env.local and fill in your credentials:
   - WHATSAPP_ACCESS_TOKEN (from Meta)
   - WHATSAPP_PHONE_NUMBER_ID (from Meta)
   - WHATSAPP_VERIFY_TOKEN (you choose this)
   - OPENAI_API_KEY (from OpenAI)
   - NEXT_PUBLIC_SUPABASE_URL (from Supabase)
   - NEXT_PUBLIC_SUPABASE_ANON_KEY (from Supabase)
   - SUPABASE_SERVICE_ROLE_KEY (from Supabase)

2. SETUP DATABASE:
   • Go to your Supabase project
   • Open SQL Editor
   • Copy all content from: supabase/schema.sql
   • Paste and click "Run"
   • This creates tables and enables realtime

3. START DEVELOPMENT SERVER:
   npm run dev
   
   Open: http://localhost:3000
   • Will redirect to /auth
   • Create an account
   • Dashboard will load at /dashboard

4. TEST APPLICATION:
   • Sign up with any email and password
   • Dashboard shows no conversations (normal - need WhatsApp messages)
   • Once you send a WhatsApp message, conversation will appear

5. CONFIGURE META WEBHOOK (when ready):
   In Meta App Dashboard:
   • Set Webhook URL: https://yourdomain.com/api/webhook
   • Set Verify Token: The WHATSAPP_VERIFY_TOKEN from .env.local
   • Subscribe to: messages event
